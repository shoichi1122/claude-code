# 📋 今日の予定まとめアプリ

その日の予定とTo Doを登録・管理して、まとめをワンクリックでコピーできるシンプルなWebアプリです。
今後の予定も日付ごとに一覧でき、Googleカレンダーから今後1年分の予定をまるごと取り込むこともできます。

## 使い方

公開版はこちら: **https://shoichi1122.github.io/claude-code/** — 開くだけで使えます。

ローカルで動かす場合、手入力だけなら `index.html` をブラウザで開くだけで動きます。
**Googleカレンダー連携を使う場合は、ローカルサーバー経由で開いてください**(Googleの認証が `file://` では動作しないため):

```bash
cd daily-schedule-app
python3 -m http.server 8000
# → ブラウザで http://localhost:8000 を開く
```

## 機能

- **予定の追加**: 日付・時刻・タイトルを入力して登録(時刻順に自動ソート、終日予定は先頭に表示)
- **今後の予定**: 明日以降の予定を「明日」「3日後」「2週間後」「3か月後」のように日付ごとにグループ表示
  - 直近2週間分を展開表示し、それより先は「すべて表示」ボタンで開閉できます
- **Googleカレンダー連携**: ボタン一つで今後1年分の予定を取り込み(下記セットアップ参照)
  - 取り込んだ予定には `GCal` バッジが付きます
  - 再読み込みしても重複せず、カレンダー側の日付・時刻・タイトル変更が反映されます(完了チェックは保持)
  - 終日予定は「終日」として各日の先頭に表示されます
- **サマリー表示**: 今日の予定数・完了数・残り数・今後の予定数と「次の予定」を常に表示(今日の予定が終わると、次は明日以降の予定を案内)
- **現在の予定をハイライト**: いま進行中の予定が青枠で強調表示(1分ごとに自動更新)
- **完了チェック**: 今日の予定はチェックボックスで完了管理
- **To Doリスト**: 時間の決まっていない「やること」をチェックリストで管理
  - 未完了のTo Doは翌日に自動で持ち越され、完了済みは日付が変わると消えます
- **まとめをコピー**: 今日の予定+To Do+今後の予定(直近1週間分)をテキスト化してクリップボードへ(SlackやLINEにそのまま貼り付け可能)

  ```
  📋 7月3日(金) の予定(1/3 完了)
  ✅ 09:00 朝会
  ⬜ 13:00 チームミーティング
  ⬜ 18:00 ジム

  ✅ To Do(1/2 完了)
  ✅ 領収書を提出
  ⬜ メール返信

  🗓 今後の予定
  ▼ 7月4日(土)
  ・終日 出張
  ・10:00 クライアント訪問
  ```

- **自動保存**: ブラウザの localStorage に保存。日付が変わると過ぎた予定は自動でクリアされ、明日の予定が「今日の予定」に繰り上がります
- **ダークモード対応**: OSの設定に自動で追従

## Googleカレンダー連携のセットアップ(初回のみ)

Googleの仕様上、カレンダーを読み取るには自分用の「OAuth クライアントID」が必要です。
無料で、5分ほどで作成できます。

1. [Google Cloud Console](https://console.cloud.google.com/) にアクセスし、プロジェクトを作成(既存のものでも可)
2. 「APIとサービス」→「ライブラリ」で **Google Calendar API** を検索して有効化
3. 「APIとサービス」→「OAuth 同意画面」を設定
   - User Type は「外部」、テストユーザーに自分のGmailアドレスを追加
4. 「APIとサービス」→「認証情報」→「認証情報を作成」→「OAuth クライアントID」
   - アプリケーションの種類: **ウェブアプリケーション**
   - 承認済みの JavaScript 生成元: `https://shoichi1122.github.io`(公開版を使う場合)/ `http://localhost:8000`(ローカルの場合)
5. 発行された `xxxx.apps.googleusercontent.com` 形式のIDをコピー
6. アプリの「⚙ 設定」を開き、クライアントIDを貼り付けて保存
7. 「🗓 Googleカレンダーから読み込み」を押すと、Googleのログイン画面が開き、許可すると今後1年分の予定が取り込まれます

### プライバシーについて

- 権限は **読み取り専用**(`calendar.readonly`)のみを要求します。カレンダーへの書き込みは行いません
- 取得した予定・クライアントIDはすべてブラウザ内(localStorage)に保存され、外部サーバーには送信されません
- 通信先は Google の認証・カレンダーAPIのみです

## 技術構成

- 単一の HTML ファイル(HTML / CSS / Vanilla JS)
- Google連携は [Google Identity Services](https://developers.google.com/identity/oauth2/web/guides/overview) + Calendar REST API(連携ボタンを押したときだけ読み込み)
- ビルド不要、外部ライブラリのバンドルなし
